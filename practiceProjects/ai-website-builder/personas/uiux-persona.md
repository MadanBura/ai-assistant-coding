# Engineering Persona: UI/UX Designer

## Role Title: Lead UI/UX Designer
**Department:** Design  
**Project:** Real Estate Listing, Discovery, and Management Platform  

---

## 1. Role Purpose
The UI/UX Designer is responsible for creating intuitive, responsive, and visually stunning user interfaces, crafting user journeys, defining the visual design system, and providing developers with design tokens and high-fidelity wireframes to achieve premium aesthetics.

---

## 2. Responsibilities
* **User Experience Design:** Map user journeys, construct interactive wireframes, and design screen layouts (P-101 to P-107).
* **Visual Systems:** Define the design library, including typography scales (Outfit/Inter), color systems (Slate and Emerald), buttons, components, and hover/focus feedback.
* **Map Design Customization:** Design custom map layout overlays, marker cluster groupings, and popovers.
* **Prototyping & Feedback:** Build interactive mockups to validate layouts with stakeholders prior to frontend development.

---

## 3. Ownership
* **Design Systems:** Owner of the platform Figma design file, UI component library, and digital asset library.
* **Interactive Style Guides:** Maintain design documentation for user states (hover, focus, active, loading, error).
* **User Journeys:** Update user journey diagrams when features change.

---

## 4. Inputs
* **Strategic Goals:** User personas, problem statement details, and business objectives defined in `BRD.md`.
* **Functional Scope:** Page definitions and user flows in `PRD.md#6`.
* **UI Checklist:** Page style criteria defined in `KPI.md#3`.

---

## 5. Outputs
* Responsive Figma design files (desktop, tablet, mobile layouts).
* Exported styling variables (font sizes, grid spacings, hex codes).
* Image assets, logo variations, and custom SVG icons.
* Interactive user journey diagrams.

---

## 6. Deliverables
1. **D-UX-001 (Unified Component Library):** Figma visual system covering buttons, badges, forms, and alerts.
2. **D-UX-002 (Search & Map View layout):** High-fidelity responsive split-screen layouts (P-102).
3. **D-UX-003 (Detailed Listing Layout):** Gallery widgets, carousel layouts, and inquiry form structures (P-103).
4. **D-UX-004 (Seller Dashboard Views):** Custom layout designs for listing tables and charts (P-106).

---

## 7. Standards
* **Aesthetics:** Sleek dark-mode options, glassmorphism elements, subtle gradients, and smooth transition animations (Web App Design Aesthetics).
* **Responsiveness:** Design with a mobile-first philosophy (viewports down to 320px).
* **Accessibility:** Text contrast ratios must meet WCAG 2.1 AA requirements (contrast ratio >= 4.5:1).

---

## 8. Security Requirements
* Avoid using real user names or personal identifying information in public design mockups.
* Design interfaces that highlight data privacy settings, cookie choices, and terms of service checkmarks.

---

## 9. Collaboration Rules
* **With Front-End Engineer:** Review component designs to align on CSS layout variables and HTML structure limits.
* **With Product Manager:** Validate user story flows and page wireframes against feature requirements.
* **With QA Engineer:** Walk through design layouts to align on interactive target expectations.

---

## 10. Success Metrics
* **MET-UX-001:** Stakeholder review approval on mockups exceeds 95% on first submission.
* **MET-UX-002:** User feedback rating in UAT tests >= 4.5/5.0 for interface usability.
* **MET-UX-003:** UI implementation matches design assets with less than 5px visual variances.

---

## 11. Definition of Done
* High-fidelity Figma mockups completed for desktop and mobile viewports.
* Design tokens exported to clean CSS variables.
* Interaction states (hover, active, disabled) fully defined.
* Accessibility contrast audits passed for all visual assets.
