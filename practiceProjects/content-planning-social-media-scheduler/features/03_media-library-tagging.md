# Feature Specification: Media Library & Asset Tagging
## Feature ID: FEAT-401

---

## 1. Purpose
Provide a centralized cloud storage workspace where users can upload, validate, tag, search, and manage visual files (images and videos). These assets are later attached to scheduled posts. Upload is performed directly to Amazon S3 to reduce web server load.

---

## 2. User Stories
* **US-401:** As an Editor, I want to upload files (JPEG, PNG, MP4) directly into a media library, categorize them with tags, and search for them during post creation.
* **US-402:** As an Editor, I want to view file validation details (file size, format, image resolution, video duration) before uploading to verify platform compatibility.

---

## 3. Functional Requirements
1. **FR-401-1:** Users MUST upload files directly from their browser client to AWS S3 using secure pre-signed upload URLs generated by the API.
2. **FR-401-2:** The frontend client MUST execute a pre-flight metadata check prior to initiating S3 uploads.
3. **FR-401-3:** File assets must have a maximum size limit enforced:
   * Images: Max 10MB (formats: JPEG, PNG, WEBP).
   * Videos: Max 150MB (formats: MP4, MOV).
4. **FR-401-4:** Upon S3 upload success, the client must trigger an API callback to register the media metadata (filename, size, MIME type, dimensions, and S3 URL key) in the PostgreSQL database.
5. **FR-401-5:** Users MUST be able to apply custom string tags (comma-separated arrays) to assets to categorize them.
6. **FR-401-6:** The system MUST support paginated search queries (default: 24 items per page) filtered by filename, tag, or MIME type group (e.g. image vs. video).
7. **FR-401-7:** Deleting an asset from the media library must remove the database record and queue a background task to delete the file object from AWS S3.

---

## 4. Validation Rules
* **MIME Types Allowed:** `image/jpeg`, `image/png`, `image/webp`, `video/mp4`, `video/quicktime`.
* **Tag Format:** Lowercase alphanumeric strings, 1-20 characters per tag, max 10 tags per asset.
* **Aspect Ratio Checks:** Warn users if an uploaded image does not meet common ratios (e.g. Instagram recommendation: 1:1, 4:5, or 16:9).

---

## 5. Edge Cases
* **S3 Upload Timeout/Interruption:** If the S3 upload fails or times out, the client must notify the user. No orphan database references should be left in PostgreSQL.
* **Same Filename Uploaded:** If an asset with an identical name is uploaded, generate a unique S3 key using a UUID prefix (e.g., `workspace_id/assets/uuid_filename.png`) to prevent overwriting existing assets.
* **Deleting Active Media Assets:** If a user attempts to delete a media asset currently attached to a scheduled or approved post (status: `PENDING_REVIEW`, `APPROVED`), block the deletion and display an alert dialog: *"This asset is currently in use by an active scheduled post and cannot be deleted until the post is published or deleted."*

---

## 6. Dependencies
* **AWS S3 Bucket:** Access to bucket credentials with read/write configuration permissions.
* **Pre-signed URL Generator:** Web application server API needs AWS SDK integration.

---

## 7. API Requirements

### 7.1 Request Pre-signed S3 Upload URL
* **POST `/api/v1/media/presign`**
* **Headers:** `Authorization: Bearer <JWT>`, `X-Workspace-ID: <workspace_id>`
* **Request Body:**
  ```json
  {
    "filename": "product_promo.mp4",
    "mime_type": "video/mp4",
    "file_size_bytes": 45120300
  }
  ```
* **Response `200 OK`:**
  ```json
  {
    "upload_url": "https://creatorsuite-media.s3.amazonaws.com/workspace_id/assets/uuid_product_promo.mp4?AWSAccessKeyId=...",
    "s3_key": "workspace_id/assets/uuid_product_promo.mp4"
  }
  ```

### 7.2 Register Uploaded Asset
* **POST `/api/v1/media`**
* **Headers:** `Authorization: Bearer <JWT>`, `X-Workspace-ID: <workspace_id>`
* **Request Body:**
  ```json
  {
    "filename": "product_promo.mp4",
    "s3_key": "workspace_id/assets/uuid_product_promo.mp4",
    "mime_type": "video/mp4",
    "file_size_bytes": 45120300,
    "tags": ["promo", "summer2026"]
  }
  ```
* **Response `201 Created`:**
  ```json
  {
    "asset_id": "c3b5d27a-8f9e-4b6a-8b83-a75d1f8876c1",
    "filename": "product_promo.mp4",
    "url": "https://d123.cloudfront.net/workspace_id/assets/uuid_product_promo.mp4",
    "tags": ["promo", "summer2026"],
    "created_at": "2026-06-04T16:20:00Z"
  }
  ```

### 7.3 Delete Media Asset
* **DELETE `/api/v1/media/:asset_id`**
* **Headers:** `Authorization: Bearer <JWT>`, `X-Workspace-ID: <workspace_id>`
* **Response `200 OK`:**
  ```json
  {
    "message": "Asset successfully removed from media library"
  }
  ```

## 8. Database Impact
Direct writes to `MEDIA_ASSET` table:

```sql
CREATE TABLE media_asset (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    workspace_id UUID NOT NULL REFERENCES workspace(id) ON DELETE CASCADE,
    filename VARCHAR(255) NOT NULL,
    s3_key TEXT NOT NULL UNIQUE,
    mime_type VARCHAR(100) NOT NULL,
    file_size_bytes INTEGER NOT NULL,
    tags TEXT[] DEFAULT '{}',
    created_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP
);

CREATE INDEX idx_media_workspace ON media_asset(workspace_id);
CREATE INDEX idx_media_tags ON media_asset USING gin(tags);
```

## 9. UI Components
* **Upload Drop Zone:** Drag-and-drop box displaying drag state overlays and progressive progress bars for active uploads.
* **Media Grid:** Resilient gallery rendering image aspect crop cards and video play icons. Click updates detailed preview side drawer.
* **Tag Input Editor:** Inline tag chips displaying simple delete close (X) controls.

---

## 10. Security Requirements
1. **Access Isolation:** CloudFront or S3 bucket security must mandate token access verification. Signed URLs or IAM policy keys ensure assets are strictly workspace-isolated.
2. **Path Sanitization:** The S3 directory path prefix MUST be structural (e.g., `workspace_id/`), preventing path traversal injections.
3. **Malicious Code Defense:** All uploaded files should pass basic MIME signature validation to verify that files match extensions (e.g., no executables hidden inside `.png`).

---

## 11. Acceptance Criteria
* **AC-401-1:** Client generates AWS S3 pre-signed upload URLs and successfully completes the file binary transmission directly to AWS S3.
* **AC-401-2:** Files matching forbidden extensions or dimensions exceed parameters are immediately aborted and report diagnostic toast alerts.
* **AC-401-3:** Deleting a media asset attached to an active scheduled post fails, returning a `409 Conflict` server code.

---

## 12. Definition of Done (DoD)
1. **Infrastructure Integration:** Successful S3 upload integration verified in staging sandbox environments.
2. **Error Safety:** Upload timeout handlers run robust client recovery workflows.
3. **Tests:** Coverage testing tracks database schema operations (GIN indexing lookup performance).
